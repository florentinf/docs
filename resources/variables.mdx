You can use variables in Neon embeds to show live info about users, servers, or music. Hereâ€™s a quick guide for each type.

## Normal Variables

| Variable                | Explanation                  |
| ----------------------- | ---------------------------- |
| `{user.id}`             | User's Discord ID            |
| `{user.tag}`            | Username#1234                |
| `{user.username}`       | Username only                |
| `{user.mention}`        | Mentions the user            |
| `{user.avatar}`         | User's avatar URL            |
| `{user.nickname}`       | User's server nickname       |
| `{user.rolehex}`        | User's top role color (hex)  |
| `{user.createdat}`      | When the user joined Discord |
| `{server.id}`           | Server's ID                  |
| `{server.name}`         | Server name                  |
| `{server.membercount}`  | Total members                |
| `{format.membercount}`  | Member count, formatted      |
| `{server.botcount}`     | Number of bots               |
| `{server.humancount}`   | Number of humans             |
| `{format.humancount}`   | Human count, formatted       |
| `{server.icon}`         | Server icon URL              |
| `{server.rolecount}`    | Number of roles              |
| `{server.channelcount}` | Number of channels           |
| `{server.ownerid}`      | Server owner's ID            |
| `{server.owner}`        | Server owner's tag           |
| `{server.createdat}`    | When the server was created  |

## Last.fm Embed Variables

| ---------------------- | --------------------------- |

| `{user.properplays}`   | Total plays (formatted)     |
| `{user.url}`           | Last.fm profile link        |
| `{author.mention}`     | Mentions the command author |

| `{author.nickname}`    | Author's nickname           |
